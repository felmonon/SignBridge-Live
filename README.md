# SignBridge Live

SignBridge Live is a single-repository web application for live sign-language translation with AI speech output.

- **Frontend**: React + Vite
- **Backend**: FastAPI
- **Translation**: Google Gemini multimodal video understanding
- **Voice output**: OpenAI `gpt-4o-mini-tts`

The app captures short live webcam clips in the browser, translates signed utterances into English, suppresses obvious duplicate outputs on the backend, and can speak the translated English aloud.

For serverless deployments, SignBridge Live can persist short session context in Upstash Redis so duplicate suppression and transcript continuity survive cold starts and multi-instance routing.

## What This Project Is

This is an end-to-end prototype for live webcam-based sign-language interpretation.

It is designed to be:

- visually polished
- simple to run locally
- easy to extend
- structured as one app in one repository

## What This Project Is Not

This is not a dedicated ASL sequence model or interpreter-grade production system.

It currently relies on a general multimodal model rather than:

- a custom sign-language video model
- gloss recognition
- signer-adapted fine-tuning
- a domain-evaluated translation benchmark

That means quality will vary with:

- lighting
- framing
- signing pace
- camera angle
- clip timing

## Architecture

High-level flow:

1. The browser opens the webcam.
2. The frontend records short clips with `MediaRecorder`.
3. The frontend sends clips to `POST /api/translate`.
4. The backend forwards the clip to Gemini with short recent conversation context.
5. The backend decides whether the clip should emit a new utterance.
6. The frontend displays the translated English.
7. The frontend optionally calls `POST /api/speak`.
8. The backend returns WAV audio generated by OpenAI TTS.

See [docs/architecture.md](docs/architecture.md) for details.

## Quick Start

### 1. Clone and enter the repo

```sh
git clone https://github.com/felmonon/SignBridge-Live.git
cd SignBridge-Live
```

### 2. Configure environment variables

```sh
cp .env.example .env
```

Set:

- `GEMINI_API_KEY`
- `OPENAI_API_KEY`

Optional for production-safe shared session state:

- `UPSTASH_REDIS_REST_URL`
- `UPSTASH_REDIS_REST_TOKEN`

### 3. Install backend dependencies

```sh
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 4. Install frontend dependencies

```sh
cd frontend
npm install
```

### 5. Build the frontend

```sh
npm run build
cd ..
```

### 6. Start the app

```sh
python run.py
```

### 7. Open the app

```text
http://127.0.0.1:8000
```

## Development

### Backend

Run the backend from the repository root:

```sh
source .venv/bin/activate
python run.py
```

### Frontend

Work on the frontend inside the repo:

```sh
cd frontend
npm install
npm run dev
```

When you want the backend-served version, rebuild:

```sh
npm run build
```

The backend serves `frontend/dist` by default.

## Environment Variables

| Variable | Required | Purpose |
| --- | --- | --- |
| `GEMINI_API_KEY` | Yes | Enables Gemini translation |
| `OPENAI_API_KEY` | Yes | Enables OpenAI TTS |
| `APP_NAME` | No | App title in backend responses |
| `HOST` | No | Bind host, default `127.0.0.1` |
| `PORT` | No | Bind port, default `8000` |
| `FRONTEND_DIST_DIR` | No | Override the default `frontend/dist` path |
| `GEMINI_MODEL` | No | Default `gemini-2.5-flash` |
| `OPENAI_TTS_MODEL` | No | Default `gpt-4o-mini-tts` |
| `OPENAI_TTS_VOICE` | No | Default `coral` |
| `CLIP_SECONDS` | No | Default clip length returned at session bootstrap |
| `SIGN_LANGUAGE_HINT` | No | Default sign-language hint, usually `ASL` |
| `SESSION_CONTEXT_LIMIT` | No | Number of recent emitted utterances forwarded back into Gemini, default `6` |
| `SESSION_TTL_SECONDS` | No | Sliding expiration for stored session context, default `1800` |
| `UPSTASH_REDIS_REST_URL` | No | Enables shared session storage for serverless deployments |
| `UPSTASH_REDIS_REST_TOKEN` | No | Enables shared session storage for serverless deployments |

## Security

- Do not commit `.env`.
- Rotate API keys if they were exposed in chat, screenshots, terminal history, or logs.
- This app sends video clips to external providers for translation and speech generation.
- For Vercel or other serverless platforms, configure Upstash Redis if you need session continuity across instances.

## Notes

- The app is clip-based, not frame-by-frame streaming.
- The backend now serves the in-repo frontend build.
- The frontend and backend now live in one repository and ship together.
